## 1. Harness & Eval Patterns

**Golden sets and regression gates are the consensus practice for LLM products in 2025-26:**
- Start with ~100 hand-curated golden examples, scale to ~500 max; quality beats auto-generated volume ([Future AGI golden-set guide](https://futureagi.com/blog/llm-eval-golden-set-design-2026/), [Pragmatic Engineer evals guide](https://newsletter.pragmaticengineer.com/p/evals)).
- Run two layers: component-level "unit" evals plus end-to-end regression over the full set on every prompt/model/architecture change; gate CI on **per-bucket** thresholds (e.g., nexus questions vs. rate lookups vs. exemption questions), not aggregate scores — aggregate gates hide category-specific regressions ([DeepEval eval-driven development](https://deepeval.com/blog/eval-driven-development), [Braintrust LLM evaluation guide](https://www.braintrust.dev/articles/llm-evaluation-guide)).
- Version the golden set diff-ably so a score drop is attributable to model vs. rubric vs. dataset change; feed every production failure back into the golden set immediately; refresh monthly from production sampling ([Galileo on static-eval failure modes](https://galileo.ai/blog/beyond-golden-datasets-static-evals-failures), [Statsig golden datasets](https://www.statsig.com/perspectives/golden-datasets-evaluation-standards)).

**LLM-judge vs deterministic grading:**
- Deterministic grading wherever criteria are objective (correct rate, correct citation, correct form field); LLM-judge only for subjective qualities (tone, completeness of a memo). Judges have systematic biases — favor longer answers, self-preference, stylistic fluency over factual accuracy — so the modern pattern is **analytic rubrics** scored criterion-by-criterion rather than one holistic score, enabling root-cause on regressions ([Monte Carlo LLM-as-judge best practices](https://montecarlo.ai/blog-llm-as-judge/), [Future AGI on judge failure modes](https://futureagi.com/blog/llm-as-a-judge/), [self-preference bias study](https://arxiv.org/pdf/2604.06996)).

**What legal/tax/medical teams actually do — and the cautionary evidence:**
- Stanford's "Hallucination-Free?" study (Journal of Empirical Legal Studies, 2025) found 17% hallucination for Lexis+ AI, ~33% for Westlaw AI-Assisted Research despite vendor "hallucination-free RAG" marketing — RAG reduces but does not eliminate error, and vendors made claims with no empirical evidence ([paper](https://onlinelibrary.wiley.com/doi/full/10.1111/jels.12413), [Stanford PDF](https://law.stanford.edu/wp-content/uploads/2024/05/Legal_RAG_Hallucinations.pdf)). An AI Hallucination Cases database now tracks ~600 court proceedings involving unverified AI output ([AI Law Librarians](https://www.ailawlibrarians.com/2026/02/19/what-the-science-says-about-hallucinations-in-legal-research/)). Tools also reinforce user errors when prompts contain false premises — evals must include adversarial/false-premise cases.
- For legally sensitive correctness, the field is converging on **neuro-symbolic split**: LLM for language/orchestration, deterministic code for anything numeric or boolean. Tax law is boolean/deterministic; LLMs are probabilistic — a hallucinated deduction cascades through downstream calculations ([Veriprajna neuro-symbolic whitepaper](https://veriprajna.com/whitepapers/ai-tax-compliance-crisis-neuro-symbolic-solution), [AWS deterministic-models case study](https://aws.amazon.com/blogs/machine-learning/overcoming-llm-hallucinations-in-regulated-industries-artificial-geniuss-deterministic-models-on-amazon-nova/)).
- Rule of thumb from practice: **tool-calling into deterministic engines** for rates, calculations, date math, jurisdiction determination; **RAG with verbatim citation** for statutory/guidance questions (increasingly "RAG 2.0" — structured queries into a verified knowledge graph, not just embedding search); **prompt-stuffing** only for small, stable, versioned context (e.g., a company's tax profile) ([RAG vs tool use guide](https://medium.com/@sumeet.pardeshi.online/rag-vs-tool-use-for-llms-the-ultimate-guide-to-choosing-the-right-approach-in-2025-c5c06b0922f6), [Determinism where it matters](https://medium.com/@codecentre76/determinism-where-it-matters-llms-where-they-help-b92e8d83db41)).

## 2. Community-Built Knowledge

**What works:**
- **Stack Overflow's tiered-privilege reputation ladder**: moderation rights unlock gradually (15 rep = flag, 2,000 = edit others' posts, 3,000 = vote to close, 10,000 = mod tools). Cost of moderation is amortized across thousands of small trusted-user actions; the system is "mostly self-regulating" ([A Theory of Moderation](https://stackoverflow.blog/2009/05/18/a-theory-of-moderation/), [Harvard D3 case study](https://d3.harvard.edu/platform-digit/submission/stack-overflow-quality-through-community-control/)).
- **Wikidata/Wikipedia**: elected admins + ML vandalism-detection (ORES-style scoring flags suspect edits so humans review a small fraction) + semi-protection on high-traffic items ([Vandalism Detection in Wikidata, CIKM](https://dl.acm.org/doi/10.1145/2983323.2983740), [Wikimedia anti-vandalism design](https://design.wikimedia.org/blog/2020/07/30/content-moderation-anti-vandalism-wikipedia.html)).
- **OpenStreetMap**: proactive moderation (moderators embedded in discussions, de-escalation first) and local-knowledge validation norms ([OSM moderation process](https://wiki.openstreetmap.org/wiki/Etiquette/Process_for_Moderation), [OSM forum on motivating accurate local contributions](https://community.openstreetmap.org/t/how-to-motivate-more-accurate-local-contributions-in-osm/130595)).
- Effective incentives: public reputation/badges tied to *specific valued behaviors* (first PR, documentation, mentoring), visible attribution, and unlocking real capability rather than just cosmetic points ([Open Source Way on incentives](https://guidebook.theopensourceway.org/guiding-participants/incentivizing-and-rewarding-participants)).

**What fails:**
- Pure gamification: leaderboards demotivate mid-tier contributors; badge-chasing optimizes for the badge, not community health ([gamification analysis](https://gamification-in-software-development.pages.dev/gamifying-open-source)).
- Moderation-as-policing: Stack Overflow's decline is partly attributed to gatekeeping culture — moderation drifted to telling posters they're wrong instead of improving posts ([Medium critique](https://medium.com/codex/its-time-for-stack-overflow-to-address-its-moderation-problems-6111e45b6e94)).
- Prepublication review of *all* edits kills contribution volume; small communities can't staff vandalism patrol without automation ([Wikipedia prepublication moderation study](https://arxiv.org/pdf/2202.05548)).
- Key lesson for a tax knowledge base: attention-seeking vandalism is rare in professional niches, but **confidently wrong drive-by edits are not** — you need review gates on high-blast-radius items (like Wikidata semi-protection of most-used items) and provenance on every claim.

## 4. Agentic Workflows in Production Fin/Tax/Legal

- **Harvey (legal, $11B, 1,800+ in-house teams)** is the reference pattern: Workflow Agents built via natural-language/visual builder with conditionals, classification, and **role-based permissions**; 500+ prebuilt use-case agents rolled out gradually via early access; "Exact Quote" **character-level citations** so every output is verifiable; sentence-level citations in research. Deliberately mirrors lawyers' existing habits of decomposing work into verifiable, citable steps ([Harvey platform](https://www.harvey.ai/platform), [Law.com on Agent Builder](https://www.law.com/legaltechnews/2026/05/05/harvey-launches-pre-built-ai-agents-self-service-customization-tool/), [GC AI review](https://gc.ai/blog/harvey-legal-ai-review)).
- **HITL as architecture, not afterthought**: structured intervention points at predetermined risk thresholds — routine steps automated, high-stakes steps (filing submission, remittance, external communication) require approval ([Galileo HITL oversight](https://galileo.ai/blog/human-in-the-loop-agent-oversight)). Majority of CFOs require human oversight of agentic AI; leading finance teams treat governance (checkpoints, threshold rules, named workflow owners) as infrastructure built *before* auditors ask ([Anrok governance frameworks](https://www.anrok.com/resources/governance-frameworks-for-agentic-ai-in-finance)).
- **Audit trails**: tamper-evident logs of inputs, outputs, decisions, and reviewers with timestamps; EU AI Act (Aug 2026 deadline) makes demonstrable human oversight and traceability a legal requirement for high-risk systems; FINRA issued agentic-AI classification guidance in 2026 ([MightyBot on audit trails](https://mightybot.ai/blog/what-are-ai-agent-audit-trails/), [Zylos governance research](https://zylos.ai/research/2026-05-01-ai-agent-governance-compliance-2026/), [Lightico on FINRA](https://www.lightico.com/blog/finra-agent-governance-banks/)).
- **TR ONESOURCE Indirect Compliance AI** ships exactly this shape in tax: rules-based orchestration + agentic assistance + HITL controls + variance explanations + complete audit documentation, explicitly marketed as not-black-box ([product page](https://tax.thomsonreuters.com/en/products/onesource-indirect-compliance-ai)).

## Implications for an indirect-tax vertical AI

**Do:**
- Split the architecture: deterministic engines (rates, sourcing, date math, nexus thresholds) behind tool calls; RAG with verbatim, pinpoint citations for guidance/statute questions; never let the model compute a liability.
- Build the eval harness before features: 100-500 golden Q&A/scenario pairs bucketed by task type (rate lookup, exemption, nexus, audit response), per-bucket CI gates, deterministic graders for anything with a right answer, rubric-based LLM-judge only for memo quality.
- Include false-premise and adversarial cases in the golden set — tools that reinforce a user's wrong assumption are a documented failure mode in tax/legal AI.
- Ship citations at the sentence/quote level with links to source (Harvey's "Exact Quote" pattern); make "no answer without a citation" a hard product rule.
- Gate irreversible actions (filing, remittance, sending anything to an auditor) behind explicit human approval with a named owner; log every agent step tamper-evidently from day one — audit-defensibility is itself the underserved market wedge.
- For community-curated tax knowledge: provenance on every claim, tiered trust (new contributors' edits reviewed; earned privileges unlock direct edit), protection on high-blast-radius entries, ML/heuristic flagging so reviewers only see suspect changes.
- Target the gap: in-house indirect-tax planning + audit response workspace, not another SMB filing autopilot.

**Don't:**
- Don't market "hallucination-free RAG" — Stanford showed 17-33% hallucination in the best-funded legal RAG tools; claim and measure a rate instead.
- Don't gate CI on a single aggregate eval score; don't treat the golden set as static.
- Don't rely on LLM-judge for correctness-critical grading (bias toward verbosity, self-preference).
- Don't use leaderboard gamification as the primary contribution incentive; don't let moderation culture become gatekeeping; don't require pre-publication review of every edit (kills volume) — review selectively by risk.
- Don't launch autonomous filing/remittance agents — that's the highest-flagged-risk category in finance agentic AI right now; earn autonomy per-step with eval evidence.
